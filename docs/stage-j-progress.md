# 阶段 J：OPC 需求广场进度

状态：功能开发完成，已进入最终回归与预览交付。

## 已交付

- 数据层：`opc_demands`、`opc_demand_connect`、`opc_demand_industries`、`demand_board_configs`，并扩展举报、收藏、通知、审计、内容指标和热榜。
- 用户端：公开列表/搜索/筛选/热门榜/详情、登录后联系方式、发布编辑、草稿与审核状态、下架/完成、对接申请/取消/状态跟踪、用户公开主页。
- 运营端：单条和批量审核、置顶/下架/封禁、审核稿编辑、全量对接记录、CSV 下载、板块配置、截止提醒和数据看板。
- 合规：普通用户每日 3 条、认证用户每日 10 条、每日最多 20 次对接；敏感金融话术、异常账号和举报均留痕并参与降权。
- 前端：需求广场、发布/编辑、详情、服务规范、用户供需工作台和运营审核台；桌面与 390px 移动视口均无页面级横向溢出。

## 验证记录

- API TypeScript、Web TypeScript、API/Web 生产构建均通过。
- API Jest：32 个套件、72 项测试通过。
- 真实 PostgreSQL 迁移、表约束和索引核对通过。
- 登录态 API：待审核列表、运营审核、联系方式、对接申请、需求方查看、状态更新、收藏、通知、数据看板、CSV 下载通过。
- 公开 API：列表、7 天热门、详情相关推荐、需求热榜通过；游客访问联系方式正确返回 401。
- 浏览器：游客隐藏联系方式；需求方能查看申请；协作者能取消并重新申请；运营账号可查看审核稿、对接记录、看板和配置；发布/编辑表单完成桌面与移动验收。

## 开发问题与处理

- TypeORM 关联分页曾使用数据库列名排序，运行时返回 500：改为实体属性名并补回归测试。
- 三处 `findOne` 缺少 `where` 导致配置、联系方式和对接接口 500：补齐条件并增加测试。
- 预览种子 SQL 两处别名错误导致事务回滚：已修正并使用容器内 UTF-8 文件执行。
- 登录回跳、公开主页 401 判断和重复对接入口在浏览器验收中暴露：已分别增加安全 `next` 回跳、结构化 `ApiError` 和有效对接状态展示。
