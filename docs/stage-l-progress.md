# 阶段 L：质量、安全与上线

更新日期：2026-07-20

## 已验证并保留的既有能力

- 用户协议、隐私政策、文章详情财经风险提示。
- JWT 访问令牌与 HttpOnly 刷新令牌、角色权限、全局及关键接口限流。
- DTO 白名单校验、上传大小限制、MIME 与文件二进制签名校验。
- 需求广场敏感词、人工审核、每日发布和对接上限。
- A–J 服务级集成测试、迁移测试与核心权限单元测试。

## 本阶段补齐

- SEO：站点级元数据、canonical、Open Graph、Twitter Card、robots、sitemap，以及资讯、政策、洞察、视频、社区和动态详情元数据。
- 合规：社区规范、版权投诉、财经免责声明；文章、视频和社区详情统一提示。
- 内容安全：纯文本规范化；论坛帖子和评论命中财经高风险词后进入人工审核队列，未审核内容不公开。
- 接口安全：更严格的刷新令牌 Cookie、密码重置后撤销刷新令牌、CORS 来源白名单、代理 IP、统一响应安全头和请求延迟/错误日志。
- 生产运行：非 root Web/API/Crawler 镜像、服务健康检查、日志轮转、Nginx 安全头与缓存、HTTPS 示例。
- 运维：PostgreSQL 每日备份/保留/恢复脚本、可选 S3 同步、数据库与磁盘监控、Webhook 告警位。
- 上线：可重复执行的发布检查、部署脚本、错误页和低风险生产冒烟测试。

## 部署边界

- 仅有公网 IP 时先提供 HTTP；配置真实域名和证书后使用 `infra/nginx.https.conf.example` 启用 HTTPS。
- 备份默认写入 Docker `postgres-backups` 卷；启用 `backup-s3` profile 前必须从环境变量提供对象存储地址与凭据。
- 发布脚本会执行迁移，但不会导入、覆盖或清理生产业务数据。
