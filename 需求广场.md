# 财经社群「OPC需求广场」需求方案
基于现有财经社群平台架构，新增**需求广场**模块，定位为OPC供需对接集市：需求方发布OPC相关需求、留下联系方式；供给方浏览需求、申请对接，实现社群内资源撮合，完全贴合现有用户、权限、社区、审核体系。

## 一、功能定位与合规底线
### 1. 核心定位
面向平台OPC从业者、产业、投资人群，搭建免费供需撮合板块：
- 需求方：发布OPC调研、数据、报告、项目落地、咨询、资料整理、实地走访等需求，可填写QQ、微信、企业联系方式；
- 供给方：浏览全部需求，点击「意向对接」发送申请，查看需求人完整联系方式；
- 平台只做信息展示，**不参与交易、不担保、不抽成**，规避居间理财、代客服务合规风险。

### 2. 强制合规规则（复用现有内容安全体系）
1. 页面统一展示免责：**本板块仅为信息交流，所有供需双方自行对接交易，平台不承担任何资金、服务纠纷责任；禁止荐股、代理财、承诺收益、内幕交易相关需求**；
2. 联系方式仅注册用户可见，游客完全隐藏；
3. 禁止发布：付费开户、带单炒股、资金募资、保本收益、违规金融中介类需求，命中直接拦截并触发人工审核；
4. 所有需求帖、联系方式经过敏感词过滤，QQ/微信/手机号增加正则校验，高频广告号直接限流；
5. 需求发布、对接记录全留审计日志，支持一键举报。

## 二、新增数据表（对接现有库结构）
### 1. `opc_demands` 需求主表
| 字段 | 说明 |
|------|------|
| id | 主键 |
| user_id | 发布人users外键 |
| title | 需求标题（限30字） |
| content | 需求详情：需要什么OPC服务、交付标准、时间、预算范围 |
| contact_info | 联系方式（QQ/微信/电话/企业微信，用户自主填写） |
| industry_category | 绑定平台现有行业分类（宏观/资本市场/科技产业等） |
| demand_type | 需求类型枚举：调研采集、报告撰写、实地走访、数据整理、政策解读、项目咨询、其他 |
| budget_range | 预算区间：无偿/500内/500-2000/2000-1w/1w以上 |
| deadline | 需求截止时间（可选） |
| status | 状态：草稿/待审核/正常展示/已对接完成/下架/违规封禁 |
| top_weight | 人工置顶权重（运营后台可调） |
| view_count | 浏览量 |
| created_at/updated_at/deleted_at | 标准时间软删除字段 |

### 2. `opc_demand_connect` 对接意向申请表
供给方点击「意向对接」生成记录
| 字段 | 说明 |
|------|------|
| id | 主键 |
| demand_id | 需求ID外键 |
| apply_user_id | 意向对接人users外键 |
| apply_msg | 对接留言（自我介绍、能提供的服务） |
| contact_status | 状态：待需求方查看、已沟通、合作完成、取消意向 |
| created_at | 申请时间 |

### 3. 关联现有表复用
- `reports`：新增需求举报类型（新增枚举：需求广告、虚假需求、违规金融信息、骚扰联系方式）；
- `moderation_logs`：记录需求审核、下架、封禁操作；
- `notifications`：新增通知类型（有人申请对接、需求审核通过/驳回、需求到期提醒）。

## 三、用户角色权限（沿用原有角色体系）
1. **游客**：仅浏览需求标题、简介，**完全隐藏联系方式**，无法申请对接、无法发布需求；
2. **注册普通用户**：
   - 发布/编辑/删除本人需求草稿；
   - 提交需求等待审核；
   - 浏览全部公开需求，查看完整联系方式；
   - 点击意向对接，发送留言；
   - 收到对接申请站内通知，查看所有意向人；
   - 标记需求「已对接完成」；
3. **认证作者/机构**：同普通用户，发布需求自带机构认证标识，排序轻微加权；
4. **版主**：对应行业分类下需求审核、下架违规需求、处理需求举报；
5. **运营/超级管理员**：全量需求管理、批量置顶、批量下架、查看全部对接记录、导出供需数据。

## 四、前端页面架构（嵌入现有网站导航）
### 4.1 顶部导航新增入口
在「社区」旁新增导航：**需求广场**，独立频道页。

### 4.2 需求广场首页（列表页）
1. 顶部操作栏
    - 发布需求按钮（未登录跳转登录）；
    - 筛选条件：需求类型、行业分类、预算范围、最新/热门/置顶、未截止需求；
    - 搜索框：搜索需求标题、内容、行业关键词。
2. 需求卡片（统一财经专业风格）
    - 发布人头像、昵称、认证标识；
    - 需求标题、行业标签、需求类型标签；
    - 需求简述、预算、截止日期、浏览量；
    - 底部按钮：查看详情、意向对接；
    - 置顶需求增加「置顶」标识，置顶卡片优先展示。
3. 侧边栏
    - 热门需求榜单（24h/7天）；
    - 发布需求快速引导；
    - 板块规则+免责声明。

### 4.3 发布需求页面
表单字段：
1. 需求标题（必填）；
2. 需求类型（下拉选择）；
3. 所属行业分类（多选）；
4. 需求详情富文本：可配图，填写OPC具体要求、交付物、时间要求；
5. 预算区间选择；
6. 需求截止时间（选填）；
7. 联系方式（必填：QQ/微信/手机号，多格式支持，前端做格式校验）；
8. 勾选同意《需求广场服务规范》；
提交逻辑：
- 提交后进入「待审核」，后台人工校验联系方式、内容合规；
- 草稿保存功能，可多次编辑再提交审核。

### 4.4 需求详情页
1. 完整需求内容、配图、行业标签、预算、截止时间；
2. 发布人完整联系方式（仅登录用户可见，游客隐藏提示「登录查看对接方式」）；
3. 操作按钮：意向对接、收藏、举报；
4. 意向对接弹窗：输入自我介绍/可提供服务留言，提交后通知需求发布人；
5. 需求发布人专属模块：展示所有意向对接人列表、对方留言、对方主页入口；
6. 底部相关推荐：同行业同类OPC需求。

### 4.5 用户中心新增模块
1. 我的发布需求：草稿/待审核/正常/已完成/已下架 分类标签；可编辑、下架、标记完成；
2. 我的对接意向：我申请对接的所有需求、对方回复状态；
3. 收到的对接申请：他人向我发起的对接记录，可查看对方主页。

## 五、后台运营管理功能（集成现有管理后台）
### 1. 需求管理列表
- 筛选：状态、发布人、行业、需求类型、发布时间；
- 批量操作：通过审核、驳回、置顶、下架、封禁；
- 单条操作：查看详情、修改内容、查看全部对接申请、查看举报记录。

### 2. 需求审核队列
左右分栏：左侧用户提交原始需求，右侧编辑修改框；
审核驳回需填写驳回理由，站内通知发送给发布用户。

### 3. 对接记录管理
全平台所有意向对接记录，支持按需求、用户筛选，导出CSV报表。

### 4. 板块配置
- 需求广场顶部Banner配置；
- 自定义置顶权重、每日置顶数量上限；
- 需求广场规范文案、免责声明配置；
- 违规关键词扩充（针对金融中介、引流、虚假收益话术）。

### 5. 数据看板新增指标
- 每日新增需求数、审核通过率、违规下架数量；
- 供需对接申请总量、成功标记完成需求数量；
- 各行业需求热度排行。

## 六、配套消息通知体系
1. 需求发布审核通过/驳回 → 通知发布人；
2. 有用户提交意向对接 → 通知需求发布人；
3. 需求截止前3天 → 提醒发布人；
4. 需求被举报、违规下架 → 通知发布人；
5. 对方查看你的对接申请 → 意向申请人通知。

## 七、热度排序逻辑（复用现有热度公式扩展）
新增需求专属热度计算，和资讯/视频逻辑统一，便于全站热榜混排：
```
需求综合热度 =
0.4 × 浏览标准分
+ 0.4 × 意向对接申请标准分
+ 0.2 × 时间新鲜度（半衰期7天）
```
防刷规则复用：同一用户多次点击对接、新账号批量发布需求、同一联系方式高频发布自动降权。

## 九、业务上线前确认参数
1. 需求发布每日条数上限（普通用户/认证用户区分）；
2. 联系方式展示规则：是否允许展示手机号，还是仅QQ/微信；
3. 无偿需求是否加权优先展示；
4. 需求置顶是否设置付费功能（三期商业化可扩展）；
5. 对接申请每日提交次数限制，防止骚扰。

## 十、扩展商业化预留（第三阶段）
1. 付费置顶需求：首页、需求广场顶部固定展示；
2. 机构认证需求专区，企业发布需求优先曝光；
3. 付费查看完整对接人联系方式（免费用户仅看昵称，付费解锁联系方式）；
4. 需求精准推送：按用户擅长行业推送匹配OPC需求。
